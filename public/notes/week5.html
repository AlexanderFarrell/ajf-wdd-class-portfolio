Exceptions are normally either
  • The System
  • Me (Programmer)
  • The user
Programmers try to limit all of these. Work with the system. Check oneself. And help the user. Exception is an error which has a return value the program can use to interpret the error.. JavaScript is pretty much the opposite of Rust in errors, whereby its super forgiving and will just keep running on most errors. Strict mode is available for JavaScript. debugger keyword adds breakpoints for you. Errors have the following:
  • Name - The error type
  • Message - What happened.
  • Stack - What led up to it
Just making a function can be a test Test Driven Development:
  1. Make Tests
  2. Write code, pass tests
  3. Refactor
  4. Test that refactored code
New features come from tests. Require things and then deliver. Jest
